# Kairo (카이로) — PRD (Product Requirements Document)

> **지식이 스스로 자라는 개인 에이전트 시스템**
> Hermes Agent의 Self-Improvement 철학에서 영감을 받아,
> 사람을 중심에 둔 "자율주행 에이전트"를 지향합니다.

---

## 1. 프로젝트 개요

### 1.1 비전 (Vision)
"에이전트가 사용자의 삶에 스며들어, 함께 성장한다."

### 1.2 미션 (Mission)
- 사용자의 일상 상호작용을 통해 스스로 진화하는 에이전트 시스템 구축
- 대화 도중 자연스러운 에이전트 생성 및 커스터마이징 지원
- 지식 조각(K-Node)이 자동으로 연결되어 새로운 인사이트를 발견

### 1.3 핵심 철학: "자율주행 에이전트"
테슬라 FSD(Autopilot)의 진화 과정에서 영감:
- FSD는 충분한 주행 데이터가 쌓여야 레벨업됨
- 마찬가지로, 에이전트도 충분한 사용자 상호작용이 쌓여야 "자율주행" 모드로 진입
- **사용자와의 상호작용 = 학습 데이터 = 성장의 원동력**

## 2. 핵심 컨셉

### 2.1 에이전트 자율주행 (Agent Autopilot) ⭐

```
Level 0: 기본 제공 에이전트만 사용 (수동 모드)
Level 1: 사용자 패턴 학습 시작 (추천 기능 활성화)
Level 2: 특정 상황 자동 대응 (e.g., "매일 아침 날씨 알림")
Level 3: 사용자 의도 예측 및 선제적 액션
Level 4: 완전 자율 — "알아서 해줘"
```

각 레벨은 사용자의 상호작용 횟수/품질에 따라 점진적으로 해금됨.

### 2.2 정적 / 동적 크론 잡 (Static & Dynamic Cron Jobs)

| 구분 | 설명 | 예시 |
|------|------|------|
| **정적 크론** | 사용자가 명시적으로 설정한 반복 작업 | "매일 오전 9시 뉴스 요약", "주간 보고서 생성" |
| **동적 크론** | 에이전트가 스스로 판단하여 생성한 작업 | "사용자가 매일 날씨를 묻네? → 아침 날씨 알림 자동 생성" |

### 2.3 빅 스킬 / 마이크로 스킬

```
┌─────────────────────────────────────────┐
│  🏗️ 빅 스킬 (Big Skill)                 │
│  • 복잡한 워크플로우                      │
│  • 여러 마이크로스킬의 조합               │
│  • 예: "여행 플래너" = 검색 + 일정 + 날씨 │
├─────────────────────────────────────────┤
│  🧩 마이크로 스킬 (Micro Skill)          │
│  • 단일 동작 단위                        │
│  • 동적 크론 잡의 실행 단위               │
│  • 예: 웹검색, PDF 요약, 번역, 알림      │
└─────────────────────────────────────────┘
```

### 2.4 지식 파이프라인 & 자율 결합

```
[문서/대화] → [청크 분할] → [임베딩] → [벡터DB 저장]
                                              ↓
                                       [LLM이 관계 감지]
                                              ↓
                              ┌────────────────┴───────────────┐
                              │                                │
                         [지식 결합 제안]                  [대기]
                              │                                │
                         [사용자 확인]                    [자동 결합]
                              │                                │
                              └────────────┬───────────────────┘
                                           ↓
                                   [새로운 합성지식 탄생]
```

## 3. 기능 요구사항

### 3.1 MVP (1주일 버전)

| 우선순위 | 기능 | 설명 |
|:---:|------|------|
| 🅿️ | **Streamlit 기반 웹 UI** | 채팅 인터페이스 + 지식 관리 대시보드 |
| 🅿️ | **RAG 기반 지식 QA** | PDF/문서 업로드 → 질문 → 답변 |
| 🅿️ | **기본 에이전트 3종** | 웹리서치, 플래너, 코딩 도우미 |
| 🅿️ | **지식 그래프 시각화** | 지식 조각 간 연결 관계 표시 |
| 🅿️ | **README + 발표 PPT** | 오픈소스 활용 과정 명시 |
| ✨ | **동적 크론 잡** | 사용자 패턴 기반 자동 작업 생성 |
| ✨ | **마이크로스킬 시스템** | 사용자 정의 가능한 스킬 |
| ✨ | **자율주행 레벨 시스템** | 상호작용 기반 레벨업 |

### 3.2 기술 스택

| 계층 | 기술 | 사유 |
|------|------|------|
| **Frontend** | Streamlit | 빠른 프로토타이핑, Python 기반 |
| **LLM** | LangChain + Jiminbox API | 쉬운 LLM 추상화, 다양한 모델 |
| **Vector DB** | ChromaDB | 가벼움, 파일 기반, 설치 불필요 |
| **Embedding** | sentence-transformers | 로컬 임베딩, 무료 |
| **지식 그래프** | NetworkX | 관계 시각화, 가벼움 |
| **크론** | APScheduler (Python) | 동적 작업 스케줄링 지원 |
| **배포** | Streamlit Cloud / Docker | 무료 배포 가능 |

### 3.3 사용 오픈소스 목록 (명시 필수)

| 오픈소스 | 라이선스 | 활용 부분 |
|----------|---------|----------|
| LangChain | MIT | LLM 호출, RAG 파이프라인 |
| ChromaDB | Apache 2.0 | 벡터 저장소 |
| Streamlit | Apache 2.0 | 웹 UI |
| sentence-transformers | Apache 2.0 | 임베딩 |
| NetworkX | BSD-3 | 지식 그래프 |
| APScheduler | MIT | 동적 크론 |
| Hermes Agent | (참고) | 자율주행/스킬/크론 **개념적 영감** |

## 6. 오픈소스 활용 전략 (평가 대비)

### "수정/업그레이드한 부분 표시" 방법
- 코드 내에 주석으로 표시: `# [KAIRO-MODIFIED] ...`
- README에 `### 변경사항` 섹션 추가
- 각 오픈소스별 "원본 기능" vs "우리가 추가한 기능" 비교표

### "협업 과정 증명" 방법
- GitHub Issues에 기능별 Issue 등록
- 브랜치 전략: `main` / `feature/*` / `docs/*`
- PR 템플릿 사용 (무슨 오픈소스를 어떻게 활용했는지 기술)

## 7. 향후 확장 (이번 프로젝트 범위 밖)

- 대화 중 자연어 → 마이크로스킬 자동 생성 (LLM이 코드 짜줌)
- 여러 사용자 간 지식 공유
- 외부 API 연동 (캘린더, 메일, 슬랙)
- 로컬 LLM 지원 (Ollama 등)
- Hermes Agent와의 실제 연동 (gateway plugin 형태)
